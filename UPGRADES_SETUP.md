# Upgrades — authoring the pool (GE content)

> **STATUS (2026-06-11):** the pool is now **35 cards** (was 8) — weapon tracks, milestones,
> hero-gated ability tracks, behavior evolutions, synergy + utility cards (D-0018/19/20).
> Bulk content is generated by `Tools/py_make_loop_v3_content.py`. This doc is kept as the
> **GE-authoring reference** for adding more cards.
>
> **Lesson learned (applies to any new GE):** for a *permanent* roguelike buff authored as an
> **INSTANT** effect, every modifier must use **ADD_BASE** (it permanently adds to the base value).
> **MULTIPLY_ADDITIVE on INSTANT multiplies the base** (e.g. MoveSpeed 600 × 0.15 = 90), which is
> almost never what you want. Use percentages only with **Infinite**-duration GEs. Flat bonuses that
> start at 0 (BarrageRadiusBonus) must be ADD_BASE regardless. A `MoveSpeed` GE also relies on the
> hero's `OnMoveSpeedChanged` callback (now wired) to take effect live.

The roguelike upgrade **flow** is built and verified (D-0018/19/20). Authoring **more** upgrade
content (the GameplayEffect assets + defs) is the part this doc covers.

## How the flow works (already wired, don't change casually)
- **Triggers:** team-XP **level-ups** (every kill feeds one squad pool; a level-up pauses the
  raid for a pick) and the Brood-mother's **upgrade chest** (synergy cards only). The old
  arena-clear offer remains as a bonus. (D-0018)
- **Rolling:** server-side, seeded (fresh stream salted by offer index + player index). Each
  player gets their **own 3-card hand**, filtered by **eligibility** — stack cap (`MaxStacks`),
  self-prereqs (milestones), squad-scope duo prereqs (synergy cards), and the hero gate
  (`RequiredHeroClass`). Short hands pad from the `UtilityPool`; one squad-shared reroll per
  raid; the watchdog auto-picks card 0 on timeout. (D-0019/20)
- **UI:** `UUpgradeSelectWidget` builds its card row at runtime on the CommonUI **GameMenu**
  layer — there is **no WBP layout to author**. Pick → `Server_ApplyUpgrade`, validated against
  the offered hand, GE applies to the picker's ASC (`bApplyToSquad` cards hit every hero's ASC).
- Each `URogueUpgradeDef` (a Data Asset) carries `DisplayName`/`Description`/`Rarity`, the
  `Effect` (`TSubclassOf<UGameplayEffect>`), and the eligibility fields above.

## What to author: the GameplayEffect assets
Make a **Blueprint Class → GameplayEffect** for each upgrade (Content/Blueprints/, prefix `GE_`).
For a permanent flat run buff set **Duration Policy = Instant** with **ADD_BASE** modifiers (see
the lesson above); use **Infinite** only for genuine percentage multipliers. Examples:

| Upgrade (example) | Attribute (`URogueHealthSet` / `URogueCombatSet`) | Op | Magnitude |
|---|---|---|---|
| Vitality | `RogueHealthSet.MaxHealth` | Add | +50 |
| Bulwark | `RogueHealthSet.Armor` | Add | +25 |
| Overshield | `RogueHealthSet.MaxShield` (+ a Shield top-up) | Add | +50 |
| **Chain Detonation** (synergy) | `RogueCombatSet.BarrageClusterBonus` | Add | +1.0 |
| **Wide Barrage** (synergy) | `RogueCombatSet.BarrageRadiusBonus` | Add | +150 |
| Power | `RogueCombatSet.AbilityPower` | Add | +0.25 |
| Cooldown | `RogueCombatSet.CooldownReduction` | Add | +0.15 |

> Behavior-changing cards mostly flip **attributes the code reads as flags/magnitudes** —
> pierce/chain/burn/poison and the D-0020 evolution attributes live on `URogueCombatSet` and are
> proc'd inside `UCombatSubsystem`. A new *number* card is pure content; a new *mechanic* card
> usually also needs a seam or kill-event hook (see D-0020 for the hybrid pattern).

## Then wire the pool
For each GE, point a `URogueUpgradeDef` at it (the `Effect` field), fill the eligibility fields
(`MaxStacks`, prereqs, `RequiredHeroClass`, `bSynergyUpgrade`, `bApplyToSquad` as applicable),
and add it to `BP_RaidGamemode.UpgradePool` (synergy cards roll only from the chest; utility
filler goes in `UtilityPool`). The 35 existing defs live under `/Game/Upgrades` and were
generated by `Tools/py_make_loop_v3_content.py` — extend that script for bulk additions.

The pick screen needs **nothing per card** — `UUpgradeSelectWidget` builds its cards at runtime
from the def's display fields.
